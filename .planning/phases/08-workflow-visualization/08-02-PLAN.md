# Plan 08-02: Workflow Panel UI — Bottom Panel, Mermaid Rendering, Auto-Update

**Phase:** 08-workflow-visualization
**Requirements:** WF-01, WF-02, WF-03
**Depends on:** 08-01 (core logic must exist)

## Goal

Build the workflow panel UI: a resizable bottom panel that renders the mermaid diagram, supports focus mode, click-to-navigate, zoom/pan, export buttons, and auto-updates on document changes.

### Wave 2: Mermaid Rendering + Auto-Update

#### Task 2.1: `useWorkflow` hook — reactive diagram generation

**File:** `src/hooks/use-workflow.ts` [NEW]

**Logic** (follows `use-preview.ts` pattern):
1. Subscribe to `useDocumentStore` for `connections`, `dataEntities`, and all pages
2. Subscribe to `useCanvasStore` for `activePageId`
3. On change (debounced 500ms):
   - Call `buildWorkflowGraph(document)`
   - If focus mode: call `filterGraphByFocus(graph, activePageId)`
   - Call `generateMermaid(graph)`
   - Return mermaid string
4. Expose: `{ mermaidText, focusMode, toggleFocus, isLoading }`

#### Task 2.2: Mermaid rendering + SVG interaction

**In:** `src/components/panels/workflow-panel.tsx` [MODIFY]

**Rendering:**
1. Call `mermaid.render('workflow-diagram', mermaidText)` → get SVG string
2. Insert SVG via `dangerouslySetInnerHTML` into container div
3. Apply CSS transform for zoom/pan state

**Click-to-navigate:**
1. Attach click handler on SVG container
2. Detect clicked node → extract node ID from mermaid SVG data attributes
3. If screen page: `useCanvasStore.getState().setActivePageId(id)`
4. If entity: navigate to ERD page

**Zoom/Pan:**
1. `onWheel` → adjust CSS `scale()` transform (0.1x → 5x range)
2. `onMouseDown + onMouseMove` → adjust CSS `translate()` transform
3. "Fit to view" button: reset scale to fit SVG in container

### Wave 3: Export + Top Bar Button + Keyboard Shortcut
