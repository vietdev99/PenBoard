    The `createDebouncedUpdater` implementation:
    ```typescript
    export function createDebouncedUpdater(callback: () => void, delayMs: number = 500) {
      let timeoutId: ReturnType<typeof setTimeout> | null = null
      let cleaned = false

      return {
        trigger: () => {
          if (cleaned) return
          if (timeoutId !== null) clearTimeout(timeoutId)
          timeoutId = setTimeout(() => {
            timeoutId = null
            callback()
          }, delayMs)
        },
        cleanup: () => {
          cleaned = true
          if (timeoutId !== null) {
            clearTimeout(timeoutId)
            timeoutId = null
          }
        },
      }
    }
    ```

Use a `Set<string>` keyed by `${entityId}:${pageId}` to deduplicate.

Deduplicate bidirectional pairs (if A->B exists, skip B->A).

    **Step 3: Create workflow-mermaid-generator.ts**

    Create `src/services/workflow/workflow-mermaid-generator.ts` with a single exported function `generateMermaidSyntax(graph: WorkflowGraph, activePageId?: string | null): string`:

    1. Start with `flowchart LR` (LR = left-to-right per research recommendation)
    2. Add classDef lines:
       - `classDef pageNode fill:#1e1e2e,stroke:#45475a,color:#cdd6f4` (card-like dark appearance)
       - `classDef entityNode fill:#313244,stroke:#45475a,color:#a6adc8,stroke-dasharray:5 5` (dashed border for entities)
       - `classDef activePage fill:#89b4fa,stroke:#89b4fa,color:#1e1e2e` (highlighted active)
    3. For each node: sanitize ID by prefixing `p_` for pages, `e_` for entities. Use `escapeLabel()` helper that replaces `"` with `#quot;` and `|` with `#124;` in Mermaid labels.
       - Page nodes: `p_ID["PageLabel"]:::pageNode` (or `:::activePage` if `node.id === activePageId`)
       - Entity nodes: `e_ID[("EntityLabel")]:::entityNode` (cylindrical shape)
    4. For each edge (using sanitized IDs):
       - connection: `p_SRC -->|"label"| p_TGT`
       - data-binding: `e_SRC -.-> p_TGT`
       - entity-relation: `e_SRC --- e_TGT`
    5. Add click directives: `click p_ID __wfNavigate` for pages, `click e_ID __wfNavigate` for entities
    6. Return joined lines with `\n`

    Also export: `buildMermaidIdMap(graph: WorkflowGraph): Map<string, string>` that returns sanitized mermaid ID -> original ID map for click callbacks.

    **Step 4: Create workflow-focus-filter.ts**

    Create `src/services/workflow/workflow-focus-filter.ts` with a single exported function `filterToFocusGraph(graph: WorkflowGraph, activePageId: string): WorkflowGraph`:

    1. Start with `neighborIds = Set([activePageId])`
    2. For each edge in graph.edges: if `edge.source === activePageId`, add `edge.target` to neighborIds. If `edge.target === activePageId`, add `edge.source` to neighborIds.
    3. Create `visibleNodeIds = Set(neighborIds)`. Then for each edge, if edge type is `data-binding` or `entity-relation`: if source is in neighborIds, add target to visibleNodeIds. If target is in neighborIds, add source to visibleNodeIds.
    4. Filter nodes: only those with id in visibleNodeIds.
    5. Filter edges: only those where BOTH source and target are in visibleNodeIds.
    6. Return filtered WorkflowGraph.
