# Plan 08-01: Core Workflow Logic — Graph Builder, Mermaid Generator, Export

**Phase:** 08-workflow-visualization
**Requirements:** WF-01, WF-02
**Depends on:** Phase 5 data binding types, Phase 2 connections

## Goal

Build the pure-logic layer that transforms PenDocument connections + data bindings + entity relations into a mermaid diagram string. No UI in this plan — just testable functions.

## Tasks

### Wave 1: Graph Builder + Mermaid Generator

#### Task 1.1: `buildWorkflowGraph()` — Graph extraction from document

**File:** `src/services/workflow/graph-builder.ts` [NEW]

```typescript
// Types
interface WorkflowNode {
  id: string
  label: string
  type: 'screen' | 'entity'
}

interface WorkflowEdge {
  source: string
  target: string
  type: 'connection' | 'data-binding' | 'entity-relation'
  label?: string // "click → push" for connections
}

interface WorkflowGraph {
  nodes: WorkflowNode[]
  edges: WorkflowEdge[]
}
```

**Logic:**
1. Walk `doc.pages[]` → create `screen` nodes for pages with `type !== 'component'`
2. Walk `doc.dataEntities[]` → create `entity` nodes
3. Walk `doc.connections[]` → create `connection` edges (`sourcePageId → targetPageId`, label = `"${triggerEvent} → ${transitionType}"`)
4. Walk all nodes recursively → find `node.dataBinding?.entityId` → create `data-binding` edges (page containing node → entity)
5. Walk `entity.fields[]` → find `type === 'relation'` with `relatedEntityId` → create `entity-relation` edges

**Tests:** `src/__tests__/workflow/graph-builder.test.ts` [NEW]
- Empty document → empty graph
- 3 pages + 2 connections → 3 nodes + 2 edges
- Data binding → dashed edge from page to entity
- Entity relation → edge between entities
- Component pages excluded from graph
- Duplicate edges deduplicated

#### Task 1.2: `generateMermaid()` — Mermaid syntax from graph

**File:** `src/services/workflow/mermaid-generator.ts` [NEW]

**Logic:**
1. Start with `graph LR` (left-to-right — fits storyboard flow naturally)
2. Emit node definitions: screens = `id["Page Name"]`, entities = `id[("Entity Name")]` (cylinder shape)
3. Emit edge definitions:
   - `connection`: `A -->|"click → push"| B`
   - `data-binding`: `A -.-> B` (dashed)
   - `entity-relation`: `A --- B` (plain)
4. Add `classDef` styles for screen vs entity nodes

Output: valid mermaid string ready for `mermaid.render()`

**Tests:** `src/__tests__/workflow/mermaid-generator.test.ts` [NEW]
- Empty graph → minimal valid mermaid
- Screen nodes use rectangle shape
- Entity nodes use cylinder shape
- Connection edges have labels
- Data binding edges are dashed
- Special characters in labels escaped

### Wave 2: Focus Filter + Export

#### Task 2.1: `filterGraphByFocus()` — 1-hop neighbor filter

**File:** `src/services/workflow/graph-builder.ts` (add to existing)

```typescript
function filterGraphByFocus(graph: WorkflowGraph, activePageId: string): WorkflowGraph
```

**Logic:**
1. Find active page node
2. Collect 1-hop neighbors (nodes connected by any edge in either direction)
3. Include data entities connected to any visible page
4. Return filtered graph with only relevant nodes + edges

**Tests:**
- 5-page graph, focus on middle → returns 3 pages (active + 2 neighbors)
- Includes entity connected to active page
- Excludes disconnected pages
- Focus on page with no connections → returns just that page

#### Task 2.2: `exportMermaid()`, `exportSVG()`, `exportPNG()` — Export utilities

**File:** `src/services/workflow/workflow-export.ts` [NEW]

- `exportMermaid(mermaidText: string)` → copy to clipboard
- `exportSVG(svgElement: SVGElement)` → download as .svg file
- `exportPNG(svgElement: SVGElement, scale?: number)` → rasterize to canvas → download .png

**Tests:**
- Mermaid export copies to clipboard (mock navigator.clipboard)
- SVG export creates download link
- PNG export creates canvas + download
