# Phase 02 — UI Design Contract

> Visual and interaction contract for Storyboard Connections & Data Entities. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope:** This phase adds three UI surfaces:
1. **Connection Section** in the property panel (CONN-01 through CONN-05)
2. **Data Entities Panel** as a floating resizable panel (DATA-01 through DATA-05)
3. **ERD Page** rendered on CanvasKit/Skia canvas (ERD-01 through ERD-04)

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (initialized) |
| Preset | new-york, neutral base, cssVariables: true |
| Component library | Radix UI (via shadcn/ui) |
| Icon library | lucide-react |
| Font | System stack: -apple-system, BlinkMacSystemFont, "Segoe UI", "Roboto", sans-serif |

**Source:** `components.json` (style: new-york, rsc: false, tsx: true, baseColor: neutral, iconLibrary: lucide). Existing `src/components/ui/` contains: Button, Select, Separator, Slider, Switch, Toggle, Tooltip.

**New shadcn components needed for this phase:**
- `Input` — for connection label editing, entity name editing, field name editing, data cell editing
- `DropdownMenu` — for ERD page creation in page tabs, field type picker, connection trigger/transition selectors
- `Dialog` — for field type change confirmation (when rows have data), delete entity confirmation
- `Badge` — for PK/FK indicators in data panel field list
- `Popover` — for filter/sort controls in data views
- `Table` — for Notion-like data row display in data panel

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon-to-label gap in connection rows, field type icon padding |
| sm | 8px | `gap-2`, `p-2`, `px-2` | Property panel section inner padding, dropdown item padding, data cell padding |
| md | 16px | `gap-4`, `p-4`, `px-4` | Data panel body padding, ERD node internal padding |
| lg | 24px | `gap-6`, `p-6` | Floating panel header padding, ERD node spacing between sections |
| xl | 32px | `gap-8` | Minimum ERD node-to-node gap |
| 2xl | 48px | `gap-12` | Not used this phase |
| 3xl | 64px | `gap-16` | Not used this phase |

Exceptions:
- **Connection section rows:** `h-7` (28px) row height, matching existing property panel row heights (SectionHeader `h-7`, Select `h-7`)
- **Data panel table rows:** `h-8` (32px) row height for comfortable inline editing
- **ERD table nodes on canvas:** 200px minimum width, scaled by zoom. Internal padding 12px (3 * 4px base).
- **Connection badge on canvas:** 16px diameter circle (scaled by `1/zoom`), matching agent badge pattern

|------|------|--------|-------------|----------------|-------|

## Component Inventory

### Surface 1: Connection Section (Property Panel)

**Location:** `src/components/panels/connection-section.tsx`
**Renders inside:** `property-panel.tsx`, below AppearanceSection, above ExportSection
**Pattern:** Follows `fill-section.tsx` — receives node data, emits updates via store

| Component | shadcn Primitive | Description |
|-----------|-----------------|-------------|
| ConnectionSection | Container `div` | Wrapper with SectionHeader |
| SectionHeader | Existing `section-header.tsx` | Title "Navigate to" with + button in actions slot |
| ConnectionRow | `div` with flex layout | One row per connection: target page name, trigger badge, transition badge, label, delete button |
| TargetPageSelect | `Select` (existing) | Dropdown of available pages (excludes current page and ERD pages) |
| TriggerSelect | `Select` (existing) | Dropdown: click / hover / submit |
| TransitionSelect | `Select` (existing) | Dropdown: push / modal / replace |
| LabelInput | `Input` (new) | Optional text input for connection label |
| DeleteButton | `Button` variant="ghost" size="icon" | Trash2 icon, 16px, `text-muted-foreground hover:text-destructive` |

**States:**
- **No connections:** Show "Navigate to" header with + button. No empty state text (consistent with fill-section pattern where no fills = just the add button).
- **Has connections:** List connection rows. Each row is collapsible or inline depending on field count.
- **Adding new:** Click + opens inline row with target page Select pre-focused.

**Interaction contract:**
- Selecting a target page creates a connection with defaults: `triggerEvent: 'click'`, `transitionType: 'push'`, `label: undefined`
- All selects update the connection in store immediately (no save button)
- Delete button removes connection with no confirmation (reversible via undo)
- Multiple connections per element are allowed (different targets)

| FieldTypeSelect | `DropdownMenu` (new) | Type picker: text, number, date, select, multi-select, checkbox, url, email, relation |

| SortRow | `div` with flex | Field select + direction toggle (asc/desc) |
| ViewTabs | `div` with tabs | Switch between views, add new view |

**Panel toggle:**
- Add `dataPanelOpen` to `canvas-store.ts` (same pattern as `variablesPanelOpen`)
- Toggle via toolbar button (Database icon) or keyboard shortcut `Cmd+Shift+D`
- Panel appears centered over canvas area

**States:**
- **No entities:** Panel body shows empty state (see Copywriting section)
- **Entity selected, no fields:** Show field list area with "Add field" button
- **Entity selected, no rows:** Show empty table with headers and "Add row" button
- **Entity selected, with data:** Show full table with inline editing

**Interaction contract:**
- Entity name: double-click to rename (same pattern as page tabs)
- Field name: click to edit inline, blur to commit
- Field type: select from dropdown, confirm if rows have data and type is incompatible
- Data cells: click to edit, Tab to move to next cell, Enter to move to next row
- Add row: click "+ New row" or press Enter in last cell of last row
- Delete entity: context menu on tab, confirmation dialog
- Delete field: click delete icon on field row, no confirmation (undo available)
- Delete row: click row delete button (trash icon at row end), no confirmation (undo available)

### Surface 3: ERD Page (Canvas Rendering)

**Location:** `src/canvas/skia/skia-erd-renderer.ts` (new file)
**Triggered by:** `page.type === 'erd'` detection in SkiaEngine render pipeline

| Visual Element | Rendering Method | Description |
|----------------|-----------------|-------------|
| ERD Table Node | CanvasKit rectangle + text | Rounded rectangle with header bar (entity name) and field list body |
| Table Header | Filled rectangle | Blue-500 header with entity name in white, 14px semibold |
| Field Row | Text line | Field name + type label + PK/FK badge, 11px normal |
| PK Badge | Small filled rounded rect | "PK" in amber-500, 8px text |
| FK Badge | Small filled rounded rect | "FK" in violet-500, 8px text |
| Relation Edge | CanvasKit path | Line from source table to target table with cardinality markers |
| Cardinality Marker | Text + line caps | "1" or "N" or "M" at each end, crow's foot notation for "many" end |
| Node Shadow | CanvasKit shadow paint | Subtle drop shadow matching canvas card aesthetics |
| Selection Highlight | CanvasKit stroke | SELECTION_BLUE (`#0d99ff`) 2px border on selected ERD node |

**ERD Node Layout (pixel values, scaled by zoom):**

```
+----------------------------------+
|  [ERD_NODE_HEADER_BG]            |
|  Entity Name (14px semibold)     |  height: 32px
|                                  |  padding: 8px 12px
+----------------------------------+
|  [ERD_NODE_BODY_BG]              |
|  PK id        number        [PK]|  row height: 24px
|     name      text               |  padding: 4px 12px
|  FK userId    relation      [FK]|
|     email     email              |
|     ...                          |
+----------------------------------+
  min-width: 200px
  border-radius: 6px
  border: 1px solid ERD_NODE_BORDER
```

**States:**
- **No entities:** ERD page shows centered empty state text on canvas (see Copywriting)
- **Entities without relations:** Table nodes render without any edges
- **Entities with relations:** Edges drawn between related tables with cardinality markers
- **Dragging node:** Node follows cursor, edges update in real time
- **Selected node:** Blue selection border, property panel shows entity info (optional, can defer)
- **Hover node:** Subtle highlight on hover

**Interaction contract:**
- Click node: select it (highlight border)
- Drag node: move node, update `erdPosition` in store
- Double-click node: switch to data panel with entity focused (open data panel if closed)
- Right-click node: context menu (rename, delete, add field)
- Canvas pan/zoom: same as design pages (space+drag, scroll wheel)
- No drawing tools on ERD page (toolbar tools disabled or hidden)

## ERD Relation Edge Rendering

### Cardinality Notation

| Cardinality | Source End | Target End | Visual |
|-------------|-----------|------------|--------|
| 1:1 | Single line | Single line | `----` with "1" label at each end |
| 1:N | Single line | Crow's foot | `----<` with "1" at source, "N" at target |
| N:M | Crow's foot | Crow's foot | `>----<` with "N" at source, "M" at target |

### Edge Routing

- Edges connect from the nearest side of source node to nearest side of target node
- Use simple Manhattan routing (horizontal + vertical segments) with rounded corners (radius 4px)
- Minimum edge offset from node: 8px
- Edge label (cardinality text) positioned 12px from each node's edge
