    **1. Create data entity store actions (`src/stores/document-store-data.ts`):**

    Follow `document-store-pages.ts` and `document-store-connections.ts` patterns EXACTLY.

    Export `createDataActions(set, get)` function. Interface:
    ```typescript
    interface DataActions {
      // Entity CRUD
      addEntity: (name: string) => string
      removeEntity: (entityId: string) => void
      updateEntity: (entityId: string, updates: { name?: string }) => void

      // Field CRUD
      addField: (entityId: string, field: Omit<DataField, 'id'>) => string
      removeField: (entityId: string, fieldId: string) => void
      updateField: (entityId: string, fieldId: string, updates: Partial<DataField>) => void
      reorderField: (entityId: string, fieldId: string, newIndex: number) => void

      // Row CRUD
      addRow: (entityId: string) => string
      removeRow: (entityId: string, rowId: string) => void
      updateRowValue: (entityId: string, rowId: string, fieldId: string, value: any) => void

      // ERD position
      updateEntityErdPosition: (entityId: string, position: { x: number; y: number }) => void

      // View management
      addView: (entityId: string, name?: string) => string
      removeView: (entityId: string, viewId: string) => void
      updateView: (entityId: string, viewId: string, updates: Partial<DataView>) => void
    }
    ```

    Implementation rules:
    - Every mutation calls `useHistoryStore.getState().pushState(state.document)` FIRST
    - Always use `state.document.dataEntities ?? []` (never assume exists)
    - `addEntity` generates id with `nanoid()`, creates entity with `{ id, name, fields: [], rows: [], views: [] }`
    - `removeEntity` ALSO removes relation fields in other entities that have `relationEntityId === entityId`
    - `removeField` ALSO removes `values[fieldId]` from all rows in the entity
    - `updateEntityErdPosition` sets `erdPosition: { x, y }` on the entity — used by ERD page drag
    - `addRow` creates `{ id: nanoid(), values: {} }`
    - `addView` creates `{ id: nanoid(), name: name ?? 'Default view', filters: [], sorts: [] }`

    Also export pure utility functions (NOT store actions — plain functions for UI use):
    ```typescript
    export function applyFilters(rows: DataRow[], filters: DataFilter[], fields: DataField[]): DataRow[]
    export function applySorts(rows: DataRow[], sorts: DataSort[], fields: DataField[]): DataRow[]
    ```
    Filter operators: `equals` (exact match), `contains` (string includes), `gt`/`lt` (numeric comparison), `isEmpty` (value undefined/null/''), `isNotEmpty` (opposite).
    Sort: compare values by field type (string comparison for text, numeric for number, date parse for date).
