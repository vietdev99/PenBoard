# Phase 07.3 Context: Tile-Based Rendering + Adaptive LOD

## Decisions

### Decision 1: Full Tile Grid (Figma-style)
**Choice:** Option A — Canvas divided into fixed grid of tiles (256×256 scene units)
**Rationale:** Uniform approach that works at all zoom levels, industry-proven by Figma
**Implications:**
- Each tile = 1 OffscreenCanvas/SkSurface, renders nodes intersecting that tile
- Node spanning multiple tiles → drawn into each tile it touches
- Visible tiles determined by simple grid math against viewport (O(1))
- Memory management: evict tiles far from viewport via LRU cache

### Decision 2: Dirty Flag Per Tile
**Choice:** Option A — Edit node → find tile(s) containing it → mark dirty
**Rationale:** Minimal re-render work, only dirty tiles re-rendered
**Implications:**
- Each tile has `dirty: boolean` flag
- `syncFromDocument` diffs changed nodes → marks affected tiles dirty
- Clean tiles → blit cached SkImage (near-zero cost)
- Dirty tiles → re-render into tile surface → cache new SkImage
- Viewport change (pan/zoom) → new tiles entering viewport may need first render

### Decision 3: FPS-Based Adaptive LOD (Game Engine Style)
**Choice:** Option C — Monitor frame time, auto-adjust render tier
**Rationale:** Self-adapting to any hardware without explicit benchmark, proven in game engines
**Algorithm:**
```
target = 16ms (60fps)
if (frameTime > 16ms for 3 consecutive frames) → decrease tier
if (frameTime < 10ms for 5 consecutive frames) → increase tier

Tiers:
  T0 (Full): drawNode() with all details
  T1 (Quick): drawNode() simplified (no shadows, blur, images → placeholder)
  T2 (Tile): simple filled rects per root frame (implemented in 07.2)
```

### Decision 4: quickReject Replaces R-tree for Rendering
**Choice:** Option B — CanvasKit native culling replaces spatial index query
**Rationale:** Tile grid + quickReject is more native than R-tree query. R-tree kept for hit-testing only.
**Flow:**
```
For each visible tile:
  1. Set canvas clip to tile bounds
  2. Iterate renderNodes in tile
  3. canvas.quickReject(nodeRect) → skip if outside clip
  4. drawNode() for passing nodes
```
**Note:** R-tree (SpatialIndex) remains for hit-testing, selection, and hover detection.

## Code Context

### Integration points
- `skia-engine.ts:render()` — main render loop, will be refactored for tile pipeline
- `skia-engine.ts:surface` — single SkSurface, will be supplemented by tile surfaces
- `skia-hit-test.ts:SpatialIndex` — R-tree kept for hit-testing only
- `skia-engine.ts:benchmarkPending` — replaced by FPS-based adaptive system
- `skia-engine.ts:visibleRenderNodes` — replaced by per-tile node lists

### Prior phase decisions (07.2)
- Frame-level LOD at zoom <30% (simple rects) → becomes T2 tier in adaptive system
- Chunked async sync → still used, tile dirty-marking hooks into sync completion
- useIdleSelector → still used for panel isolation
