**Step 2: Create `src/mcp/tools/preview.ts`** (MCP-02)

Export `handleGeneratePreview(params)`:

```typescript
interface GeneratePreviewParams {
  filePath?: string
  pageId: string
  frameId?: string
}
```

Implementation:
1. `const filePath = resolveDocPath(params.filePath)`
2. `const doc = await openDocument(filePath)` — no structuredClone needed since we don't mutate
3. Validate pageId exists: `const page = doc.pages?.find(p => p.id === params.pageId)` — throw `Page not found: ${params.pageId}` if not found
4. Page type guard per user decision: if `page.type === 'erd' || page.type === 'component'`, throw `Only screen pages are previewable. Page "${page.name}" is type "${page.type}"`
5. `const syncUrl = await getSyncUrl()` — throw `No running PenBoard instance found. Start PenBoard or the dev server, then try again.` if null
6. `const previewId = generateId()`
7. Call `generatePreviewHTML(doc, params.pageId, params.frameId ?? null, previewId)` — this may fail if preview-html-generator has browser-only deps. If import fails at compile time, fall back to a lighter approach: POST the doc to a Nitro endpoint that generates server-side. But TRY the direct import first.
8. POST to `${syncUrl}/api/preview/data` with body `{ id: previewId, html }`. Use `fetch()`.
9. Return `{ previewId, url: \`${syncUrl}/preview/${previewId}\`, pageName: page.name, pageId: params.pageId, frameId: params.frameId ?? null }`

**Step 3: Create `src/mcp/tools/workflow.ts`** (MCP-04)

Export `handleExportWorkflow(params)`:

```typescript
interface ExportWorkflowParams {
  filePath?: string
  format: 'mermaid' | 'svg' | 'png'
  focusPageId?: string
}
```

Implementation:
1. `const filePath = resolveDocPath(params.filePath)`
2. `const doc = await openDocument(filePath)`
3. `let graph = buildWorkflowGraph(doc)`
4. If `params.focusPageId`, apply `graph = filterGraphByFocus(graph, params.focusPageId)`
5. `const mermaidText = generateMermaid(graph)`
6. If `params.format === 'mermaid'`, return `{ format: 'mermaid', content: mermaidText, nodeCount: graph.nodes.length, edgeCount: graph.edges.length }`
7. If format is 'svg' or 'png', try to render:

```typescript
async function renderMermaidToBase64(definition: string, format: 'svg' | 'png'): Promise<string> {
  try {
    const { renderMermaid } = await import('@mermaid-js/mermaid-cli')
    const puppeteer = await import('puppeteer')

    if (!cachedBrowser) {
      cachedBrowser = await puppeteer.default.launch({ headless: 'shell' })
    }

    const { data } = await renderMermaid(cachedBrowser, definition, format, {
      backgroundColor: 'white',
    })

    return Buffer.from(data).toString('base64')
  } catch (err) {
    throw new Error(
      `SVG/PNG export requires @mermaid-js/mermaid-cli. Install with: bun add @mermaid-js/mermaid-cli. ` +
      `Mermaid text format is always available. Error: ${err instanceof Error ? err.message : String(err)}`
    )
  }
}
```

Use a module-level `let cachedBrowser: any = null` for Puppeteer browser caching. Clean up on process exit:
```typescript
process.on('exit', () => { cachedBrowser?.close() })
```

8. Return `{ format, content: base64String, nodeCount: graph.nodes.length, edgeCount: graph.edges.length, encoding: 'base64' }`
