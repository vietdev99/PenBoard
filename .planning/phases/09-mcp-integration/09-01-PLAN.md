**Step 3: Create `src/mcp/tools/data-binding.ts`** (MCP-01)

Export `handleManageDataBinding(params)` following the load-mutate-save pattern:

- `set_binding`: Validate nodeId and entityId are provided. Load doc, find node via `findNodeInTree(getDocChildren(doc, pageId), nodeId)`. Verify the entity exists in `doc.dataEntities`. Set `node.dataBinding = { entityId, fieldMappings: params.fieldMappings ?? [] }` via `updateNodeInTree`. Save. Return `{ ok: true, nodeId, entityId, fieldMappings }`.

- `remove_binding`: Validate nodeId. Load doc, find node. Set `dataBinding` to undefined via `updateNodeInTree`. Save. Return `{ ok: true, nodeId }`.

- `list_bindings`: Load doc. Get all nodes for the page via `getDocChildren + flattenNodes`. Filter nodes where `node.dataBinding` is truthy. Return `{ bindings: [{ nodeId, nodeName, entityId, fieldMappings }] }`.

**Step 4: Create `src/mcp/tools/context.ts`** (MCP-03)

Export `handleSetContext(params)` and `handleGetContext(params)`:

- `set_context`: If `nodeId` provided, load doc, find node, update `context` field via `updateNodeInTree`, save. If `pageId` provided (and no nodeId), find page in `doc.pages`, set `page.context = params.context`, save. Return `{ ok: true, target: nodeId ?? pageId, context: params.context }`.

- `get_context`: If `nodeId` provided, load doc, find node, return `{ nodeId, name: node.name, context: node.context ?? '' }`. If `pageId` provided, find page, return `{ pageId, name: page.name, context: page.context ?? '' }`. Throw if neither provided.

**Step 5: Create `src/mcp/tools/entities.ts`** (MCP-05 entities)

Export `handleManageEntities(params)` — dispatch on `params.action`:

- `add_entity`: Create `DataEntity` with `generateId()`, name from params (default 'Entity'), empty fields/rows/views. Push to `doc.dataEntities`. Save. Return `{ entityId, entityCount }`.

- `update_entity`: Find entity by entityId. Merge name if provided. Save. Return `{ ok: true }`.

- `remove_entity`: Find entity. CRITICAL CASCADE: (1) filter out entity from `doc.dataEntities`, (2) clean relation fields in other entities that reference this entityId: `e.fields.filter(f => f.relatedEntityId !== entityId)`, (3) walk ALL page node trees and doc.children with `clearDataBindingInTree(nodes, entityId)` to remove dangling data bindings. Copy the pure `clearDataBindingInTree` function from `document-store-data.ts` (it's a local function there, not exported — re-implement it in entities.ts). Save. Return `{ ok: true, remainingEntities }`.

- `add_field`: Validate entityId. Create `DataField` with `generateId()` + params (name, type required). Push to entity.fields. Save. Return `{ fieldId, fieldCount }`.

- `update_field`: Find entity, find field. Merge updates. Save. Return `{ ok: true }`.

- `remove_field`: Find entity, remove field from entity.fields. Also clean row values: for each row, delete `row.values[fieldId]`. Save. Return `{ ok: true }`.

- `add_row`: Validate entityId. Create `DataRow` with `generateId()`, empty values (or params.fieldValues if provided). Push to entity.rows. Save. Return `{ rowId, rowCount }`.

- `update_row`: Find entity, find row. Merge fieldValues into `row.values`. Save. Return `{ ok: true }`.

- `remove_row`: Find entity, remove row. Save. Return `{ ok: true }`.

- `list_entities`: Return `{ entities: doc.dataEntities.map(e => ({ id, name, fieldCount, rowCount, viewCount })) }`.

- `get_entity`: Find entity by entityId. Return full entity object.

- `add_view`: Create DataView with generateId(), name. Push to entity.views. Save. Return `{ viewId }`.

- `remove_view`: Find entity, remove view. Save. Return `{ ok: true }`.

- `update_view`: Find entity, find view. Merge updates. Save. Return `{ ok: true }`.

**Step 6: Create `src/mcp/tools/connections.ts`** (MCP-05 connections)
