<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: End-to-end preview verification</name>
  <what-built>
    Complete interactive preview system:
    - Preview button in editor top bar (Play icon)
    - Preview HTML generation with semantic tags, CSS variables, data binding
    - Nitro server routes (POST data, GET HTML, SSE events)
    - Navigation between screens via ScreenConnections (push/modal/replace)
    - Preview toolbar (back, breadcrumb, hotspot toggle, refresh, download)
    - Hot reload via SSE when editor document changes
    - Strict CSP meta tag blocking external resources
    - Self-contained HTML download export
  </what-built>
  <how-to-verify>
    **Setup:** Start dev server with `bun --bun run dev`. Open editor at http://localhost:3000/editor.

    **PREV-01: Preview opens in browser tab**
    1. Create or open a document with at least 2 screen pages, each containing a frame with some content (text, rectangles, images)
    2. Click the Play button in the top bar
    3. Verify: A new browser tab opens showing the current page rendered as HTML
    4. Verify: The preview toolbar appears at the top (dark theme in dark mode, light in light mode)
    5. Verify: Breadcrumb shows the current page name

    **PREV-02: Navigation between screens**
    1. In the editor, create ScreenConnections between elements across pages (e.g., a button on Page 1 connecting to Page 2 with 'push' transition)
    2. Open preview again
    3. Click the connected element in preview
    4. Verify: Push transition slides left (smooth, ~250ms)
    5. Verify: Browser URL hash changes to #page-id
    6. Click browser Back button
    7. Verify: Returns to previous page
    8. Test modal connection: verify overlay with backdrop appears, clicking backdrop closes it
    9. Toggle Hotspot mode in toolbar
    10. Verify: All clickable elements are highlighted with blue dashed outline
    11. Navigate to a page ID that does not exist (manually edit hash)
    12. Verify: "Page not found" message with Back button appears

    **PREV-03: Live sample data from bound entities**
    1. In the editor, create a table component with data binding to an entity with sample rows
    2. Open preview
    3. Verify: Table in preview shows entity field names as headers and entity row data in cells
    4. Test dropdown binding: verify select element contains entity options

    **PREV-04: Sandbox security**
    1. In preview tab, open browser DevTools (F12)
    2. Go to Console tab
    3. Verify: No CSP violation warnings (if all resources are inline)
    4. Go to Elements tab, inspect <meta http-equiv="Content-Security-Policy">
    5. Verify: Contains `default-src 'none'; script-src 'unsafe-inline'; style-src 'unsafe-inline'; img-src data: blob:; font-src data:`
    6. Try: In Console, run `fetch('https://example.com')` — should be blocked by CSP

    **Hot reload**
    1. Keep preview tab open
    2. In editor, modify some text or change a fill color
    3. Verify: Preview tab reloads within ~1 second showing updated content

    **Download HTML**
    1. Click "Download" button in preview toolbar
    2. Open the downloaded .html file directly in browser (file:// protocol)
    3. Verify: Content renders correctly without server
    4. Verify: Navigation between pages works in the downloaded file
    5. Verify: No SSE reconnection errors in console (SSE script should be stripped)
  </how-to-verify>
