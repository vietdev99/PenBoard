# Plan 07.2-02: Panel Isolation — Selective Zustand Subscriptions

<objective>
Prevent UI panels from freezing when canvas performs heavy operations. Currently, Zustand store updates from canvas (viewport batch, document sync) trigger re-renders in ALL subscriber panels even when the panel's specific data hasn't changed.
</objective>

## must_haves
- Panels use Zustand selectors to subscribe only to their relevant data slices
- Panel re-renders reduced to only when their specific data changes
- Canvas viewport updates (setViewportBatch) do NOT trigger panel re-renders
- No functional regression in panel behavior

<task id="04" title="Defer non-critical panel updates with requestIdleCallback">
<read_first>
- src/canvas/skia/skia-canvas.tsx (document store subscription, line 319-333)
</read_first>

<action>
Create a custom hook `useIdleStoreSubscription` in `src/hooks/use-idle-store.ts`:

```typescript
import { useEffect, useState } from 'react'

/**
 * Subscribe to a Zustand store with requestIdleCallback scheduling.
 * Panel updates are deferred to idle time, preventing them from competing
 * with canvas rendering for main thread time.
 */
export function useIdleSelector<T, S>(
  store: { subscribe: (fn: () => void) => () => void; getState: () => S },
  selector: (state: S) => T,
  equalityFn?: (a: T, b: T) => boolean,
): T {
  const [value, setValue] = useState(() => selector(store.getState()))
  
  useEffect(() => {
    let idleHandle: number | null = null
    const unsub = store.subscribe(() => {
      if (idleHandle) return
      idleHandle = requestIdleCallback(() => {
        idleHandle = null
        const next = selector(store.getState())
        if (!equalityFn || !equalityFn(value, next)) {
          setValue(next)
        }
      })
    })
    return () => {
      unsub()
      if (idleHandle) cancelIdleCallback(idleHandle)
    }
  }, [store, selector, equalityFn])
  
  return value
}
```

Use this hook in panels for non-critical data (e.g., navigate panel connection display, layers panel node count badges).
</action>

<acceptance_criteria>

- Hook uses `requestIdleCallback` for deferred updates
- Hook properly cleans up on unmount (`cancelIdleCallback`)
- At least one panel uses the hook for non-critical subscriptions
</acceptance_criteria>
</task>
