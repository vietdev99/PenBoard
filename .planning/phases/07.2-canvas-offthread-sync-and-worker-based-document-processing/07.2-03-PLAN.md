# Plan 07.2-03: Chunked Yield syncFromDocument

<objective>
Prevent `syncFromDocument()` from blocking the main thread for 1-2s when processing 70K+ nodes. Refactor the heaviest sub-operations into yielding async patterns that return control to the main thread between chunks.
</objective>

## must_haves
- `syncFromDocument` does not block main thread for >50ms continuously
- Panels and UI remain interactive during document sync
- Interrupted sync (document changes mid-process) handled correctly
- Initial load sync still works correctly with `zoomToFitContent()`
- No visual regression

## Tasks

<task id="01" title="Refactor flattenToRenderNodes to yield in chunks">
<read_first>
- src/canvas/skia/skia-engine.ts (flattenToRenderNodes function ~line 214, syncFromDocument ~line 534)
</read_first>

<action>
1. Create `flattenToRenderNodesChunked` as an async function that processes nodes in chunks of ~500, yielding between chunks:

```typescript
async function flattenToRenderNodesChunked(
  nodes: PenNode[],
  offsetX = 0, offsetY = 0,
  parentAvailW?: number, parentAvailH?: number,
  clipCtx?: ClipInfo,
  signal?: AbortSignal,
): Promise<RenderNode[]> {
  const result: RenderNode[] = []
  let processed = 0
  const CHUNK_SIZE = 500

  // Use a stack-based iteration instead of recursion for chunking
  const stack: Array<{nodes: PenNode[], ox: number, oy: number, pw?: number, ph?: number, clip?: ClipInfo, depth: number}> = [
    { nodes, ox: offsetX, oy: offsetY, pw: parentAvailW, ph: parentAvailH, clip: clipCtx, depth: 0 }
  ]

  while (stack.length > 0) {
    if (signal?.aborted) throw new DOMException('Aborted', 'AbortError')
    
    const frame = stack.pop()!
    for (let i = frame.nodes.length - 1; i >= 0; i--) {
      // ... same logic as flattenToRenderNodes (resolve sizing, compute height, etc.)
      // Push children frames onto stack instead of recursive call
      
      processed++
      if (processed % CHUNK_SIZE === 0) {
        // Yield to main thread
        await new Promise(resolve => setTimeout(resolve, 0))
      }
    }
  }
  return result
}
```

2. Keep the original `flattenToRenderNodes` as-is for synchronous use (initial load via `syncFromDocument`).

3. Update `syncFromDocumentAsync()` to use `flattenToRenderNodesChunked` instead of `flattenToRenderNodes`.
</action>

<acceptance_criteria>
- `skia-engine.ts` contains `flattenToRenderNodesChunked` async function
- Function yields every 500 nodes via `setTimeout(resolve, 0)`
- Function accepts `AbortSignal` for cancellation
- Original synchronous `flattenToRenderNodes` unchanged (for initial load)
- `syncFromDocumentAsync` uses chunked version
</acceptance_criteria>
</task>

```typescript
async rebuildAsync(nodes: RenderNode[], signal?: AbortSignal) {
  this.tree.clear()
  this.items.clear()
  
  const items: RTreeItem[] = []
  const CHUNK = 1000
  for (let i = 0; i < nodes.length; i++) {
    // ... same item creation logic ...
    if (i % CHUNK === CHUNK - 1) {
      if (signal?.aborted) throw new DOMException('Aborted', 'AbortError')
      await new Promise(r => setTimeout(r, 0))
    }
  }
  this.tree.load(items)
}
```
