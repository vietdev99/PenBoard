# Plan 07.2-01: Frame Thumbnail LOD + Viewport-Filtered Render List

<objective>
Eliminate render lag at zoom <30% with 500+ frames by:
1. Caching bitmap thumbnails per root frame and drawing them instead of children at low zoom
2. Using spatial index R-tree viewport query to build `visibleRenderNodes` — render loop iterates O(visible) instead of O(70K)
</objective>

## must_haves
- At zoom <30%, root frames render as cached bitmap thumbnails (single drawImage per frame)
- Render loop iterates only visible nodes from spatial index query
- Thumbnails invalidate when frame content changes
- SkImage lifecycle managed (`.delete()` on invalidation)
- No visual regression at zoom >=30%

## Tasks

<task id="01" title="Add viewport query to SpatialIndex">
<read_first>
- src/canvas/skia/skia-hit-test.ts (current SpatialIndex implementation)
</read_first>

<action>
Add `queryViewport(left, top, right, bottom)` method to `SpatialIndex` class that returns `RenderNode[]` sorted by zIndex ascending (render order):

```typescript
queryViewport(left: number, top: number, right: number, bottom: number): RenderNode[] {
  const candidates = this.tree.search({ minX: left, minY: top, maxX: right, maxY: bottom })
  candidates.sort((a, b) => a.zIndex - b.zIndex) // ascending = render order (back to front)
  return candidates.map(c => c.renderNode)
}
```
</action>

<acceptance_criteria>
- `skia-hit-test.ts` contains method `queryViewport(left: number, top: number, right: number, bottom: number): RenderNode[]`
- Method returns nodes sorted by `zIndex` ascending
- Method uses `this.tree.search()` with bounding box parameters
</acceptance_criteria>
</task>

<task id="02" title="Build visibleRenderNodes from spatial query in render loop">
<read_first>
- src/canvas/skia/skia-engine.ts (render() method ~line 783, setViewport ~line 1171)
- src/canvas/skia/skia-hit-test.ts (queryViewport from task 01)
</read_first>

<action>
In `skia-engine.ts`:

1. Add field: `private visibleRenderNodes: RenderNode[] = []`

2. In `render()` method, BEFORE the main render loop (after viewport bounds calculation ~line 816), replace direct iteration of `this.renderNodes` with spatial query:

```typescript
// Build visible list from spatial index (O(visible) instead of O(70K))
const vpSceneLeft = -this.panX / this.zoom
const vpSceneTop = -this.panY / this.zoom
const vpSceneRight = vpSceneLeft + canvasW / this.zoom
const vpSceneBottom = vpSceneTop + canvasH / this.zoom
const margin = 100 / this.zoom // margin in scene coords
this.visibleRenderNodes = this.spatialIndex.queryViewport(
  vpSceneLeft - margin, vpSceneTop - margin,
  vpSceneRight + margin, vpSceneBottom + margin,
)
```

3. Replace ALL `for (const rn of this.renderNodes)` loops in render() that have viewport culling checks with `for (const rn of this.visibleRenderNodes)` — remove the inline viewport culling checks since spatial index already filtered.

4. Keep `this.renderNodes` intact — it's still needed for syncFromDocument, connection drawing, and other non-render uses.
</action>

<acceptance_criteria>
- `skia-engine.ts` contains field `visibleRenderNodes`
- Main render loop uses `this.visibleRenderNodes` instead of `this.renderNodes`
- `spatialIndex.queryViewport()` called in render() before node iteration
- Old inline viewport culling (`rn.absX + rn.absW < vpLeft`) removed from loops that now use visibleRenderNodes
- Agent overlay loops also use visibleRenderNodes
</acceptance_criteria>
</task>

<task id="03" title="Implement frame thumbnail cache and LOD rendering">
<read_first>
- src/canvas/skia/skia-engine.ts (render() method, syncFromDocument, surface field)
- src/canvas/skia/skia-renderer.ts (drawNode, drawFullNode)
</read_first>

<action>
In `skia-engine.ts`:

1. Add thumbnail cache structures:
```typescript
/** Cached bitmap thumbnails per root frame for LOD rendering at low zoom */
private frameThumbnails = new Map<string, { image: SkImage; version: number }>()
private frameContentVersions = new Map<string, number>()
private thumbnailZoom = 0.3 // threshold: below this → show thumbnails
```

2. Add method `captureFrameThumbnail(frameRn: RenderNode)`:
- Check if frame already has valid thumbnail (version matches)
- Use `canvas.save()`, clip to frame bounds, render frame and children, `canvas.restore()`
- Capture with `surface.makeImageSnapshot(iRect)` where iRect is the frame's screen-space bounds
- Store in `frameThumbnails` Map with current version
- Return the SkImage

3. Add method `invalidateFrameThumbnail(frameId: string)`:
- If `frameThumbnails.has(frameId)`, call `.image.delete()` and remove entry
- Increment version in `frameContentVersions`

4. In `syncFromDocument()`, after `flattenToRenderNodes()` completes:
- For each root frame whose children changed → call `invalidateFrameThumbnail(frameId)`

5. In `render()` method, when zoom < `thumbnailZoom`:
- For root-level frames in visibleRenderNodes, check if thumbnail exists
- If yes: skip rendering children, draw thumbnail with `canvas.drawImageRect()`
- If no: render normally (thumbnail will be captured on next high-zoom render)
- Children nodes that belong to thumbnailed frames → skip in render loop

6. In `render()` method, when zoom >= `thumbnailZoom`:
- After rendering a root frame and its children, if thumbnail needs capture → call `captureFrameThumbnail()`

7. In `destroy()`, iterate `frameThumbnails` and `.delete()` all SkImages.
</action>

<acceptance_criteria>
- `skia-engine.ts` contains `frameThumbnails` Map and `captureFrameThumbnail` method
- At zoom < 0.3 (30%), root frames draw as bitmap images via `canvas.drawImageRect()`
- At zoom >= 0.3, frames render normally and thumbnails are captured
- `invalidateFrameThumbnail` called when frame content changes in syncFromDocument
- `destroy()` calls `.delete()` on all cached SkImages
- No memory leak: SkImage.delete() called on invalidation and destroy
</acceptance_criteria>
</task>
