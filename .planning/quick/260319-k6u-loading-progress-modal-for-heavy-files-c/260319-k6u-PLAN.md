<task type="auto">
  <name>Task 2: Compute text cache key without measurement (eliminate per-frame text measurement for cache hits)</name>
  <files>
    src/canvas/skia/skia-renderer.ts
  </files>
  <action>
The biggest single perf bottleneck: `drawTextBitmap` (lines 1362-1510+) calls `getMeasureCtx()`, sets `ctx.font`, and runs `wrapLine`/`measureText` on EVERY frame for EVERY text node — even when the result is already cached. The measurement is used to compute `renderW` and `textH` which are part of the cache key. But for cache HITS, this measurement work is wasted.

Fix: Compute a "pre-cache key" from node properties alone (no measurement needed), and only do the full measurement on cache MISS.

Refactor `drawTextBitmap` as follows:

1. Extract all the node properties needed for cache key BEFORE any measurement:
   ```typescript
   const content = ... // extract text content (existing code)
   const fontSize = tNode.fontSize ?? 16
   const fillColor = resolveFillColor(tNode.fill)
   const fontWeight = tNode.fontWeight ?? '400'
   const fontFamily = tNode.fontFamily ?? 'Inter, ...'
   const textAlign = tNode.textAlign ?? 'left'
   const lineHeightMul = tNode.lineHeight ?? defaultLineHeight(fontSize)
   const textGrowth = tNode.textGrowth
   const dpr = window.devicePixelRatio || 1
   const rawScale = this.zoom * dpr
   const scale = rawScale <= 2 ? 2 : rawScale <= 4 ? 4 : 8
   ```

2. Build a "fast cache key" from these properties + the node's WIDTH and HEIGHT (which are already known from the RenderNode):
   ```typescript
   const fastKey = `${content}|${fontSize}|${fillColor}|${fontWeight}|${fontFamily}|${textAlign}|${Math.round(w)}|${Math.round(h)}|${textGrowth ?? ''}|${scale}`
   ```

3. Check a NEW `textFastCache` Map<string, string | null> that maps fastKey -> full cacheKey (or null for "render nothing"). If hit, use the full cacheKey to look up the image in `textCache` and skip ALL measurement:
   ```typescript
   const fastHit = this.textFastCache.get(fastKey)
   if (fastHit !== undefined) {
     if (fastHit === null) return // previously determined to be empty
     const img = this.textCache.get(fastHit)
     if (img !== undefined) {
       // Draw from cache — skip all measurement
       if (img) { /* existing draw logic */ }
       return
     }
     // Full cache key known but image evicted — fall through to full path
   }
   ```

4. On cache MISS, run the full measurement (existing code), compute the full cacheKey, store `textFastCache.set(fastKey, fullCacheKey)`, and proceed with rasterization.

5. Add `textFastCache` as a private field alongside `textCache`. Evict from `textFastCache` when `clearTextCache()` is called. Cap at same size (300 entries) with LRU eviction matching the existing `textCacheOrder` pattern.

6. Also move the icon font guard checks (isIconFontFamily, isUnicodePUA) BEFORE the fast cache lookup since those are cheap string checks and should short-circuit early.

This eliminates `ctx.font = ...` assignment (expensive — triggers font loading/parsing), `wrapLine()`, and `measureText()` for every cache-hit text node, which is the majority of text nodes on every frame.
  </action>

The `render()` method in skia-engine.ts iterates `this.renderNodes` 4-8 times per frame (draw nodes, frame labels, agent node borders, agent frame glow, component badges, dim overlays, BFS rnMap building x2). For pages with 500+ render nodes this is significant.

**Fix 1: Single-pass node rendering with pre-built auxiliary lists.**

In `syncFromDocument()` (or a helper called after flattenToRenderNodes), pre-compute auxiliary lists alongside `this.renderNodes`:

```typescript
// After this.renderNodes = flattenToRenderNodes(measured)
this.rootFrameNodes = []     // RenderNodes that are root frames (no clipRect)
this.reusableNodes = []      // RenderNodes with reusable IDs
this.labelNodes = []         // RenderNodes needing frame labels (root frames + reusable)
for (const rn of this.renderNodes) {
  const isRoot = rn.node.type === 'frame' && !rn.clipRect
  if (isRoot) this.rootFrameNodes.push(rn)
  if (this.reusableIds.has(rn.node.id)) this.reusableNodes.push(rn)
  if (rn.node.name && (isRoot || this.reusableIds.has(rn.node.id))) {
    this.labelNodes.push(rn)
  }
}
// Pre-build rnMap for connection arrow lookups
this.rnMap = new Map()
for (const rn of this.renderNodes) {
  this.rnMap.set(rn.node.id, { absX: rn.absX, absY: rn.absY, absW: rn.absW, absH: rn.absH })
}
```

Then in `render()`:
- Replace the "frame labels" loop (iterates ALL renderNodes) with `for (const rn of this.labelNodes)` — skip viewport cull check only if labelNodes is small.
- Replace both `rnMap` constructions (lines ~766 and ~879) with `this.rnMap` — built once in sync, not per frame.
- Replace the "component badges" loop (iterates ALL renderNodes) with `for (const rn of this.reusableNodes)`.
- Replace the "dim overlays" loop with `for (const rn of this.rootFrameNodes)`.

**Fix 2: Cache getCanvasBackground() per frame.**

`getCanvasBackground()` in canvas-constants.ts does `document.documentElement.classList.contains('light')` which is a DOM query. It's called once per render() but could be cached. Add a module-level cache:

```typescript
let _bgCache: string | null = null
let _bgCacheTick = 0
export function getCanvasBackground(): string {
  const tick = performance.now()
  if (_bgCache && tick - _bgCacheTick < 1000) return _bgCache
  _bgCacheTick = tick
  if (typeof document === 'undefined') return (_bgCache = CANVAS_BACKGROUND_DARK)
  _bgCache = document.documentElement.classList.contains('light')
    ? CANVAS_BACKGROUND_LIGHT
    : CANVAS_BACKGROUND_DARK
  return _bgCache
}
```

This is a minor optimization but eliminates a DOM query every frame.

**Fix 3: LOD — skip text rendering when zoomed out past threshold.**

In `skia-renderer.ts` `drawNode` method, when the viewport zoom is below a threshold AND the node's screen-space size is very small, skip expensive text rendering and draw a simple colored rectangle placeholder instead.

In the `drawText` method (line 1288), add an early LOD check:
```typescript
// LOD: when node is too small on screen, draw placeholder instead of text
const screenH = h * this.zoom
if (screenH < 8) {
  // Text would be unreadable — draw a colored rect as placeholder
  const paint = new ck.Paint()
  paint.setColor(parseColor(ck, resolveFillColor((node as TextNode).fill)))
  paint.setAlphaf(opacity * 0.4)
  canvas.drawRect(ck.LTRBRect(x, y, x + w, y + Math.min(h, 6)), paint)
  paint.delete()
  return
}
```

This avoids ALL text measurement, wrapping, cache lookup, and rasterization for text nodes that would be sub-pixel anyway. At zoom 0.15 with 16px text, screenH = 2.4px — completely unreadable, so a gray bar is a perfect placeholder.

**Fix 4: Cache overlay text images in drawText2D (frame labels).**

The `drawText2D` function in skia-overlays.ts already has a cache (`textImageCache`), but `drawFrameLabelColored` calls it with a `fontSize` that varies continuously with zoom (`12 / zoom`). This means cache NEVER hits during zoom because fontSize changes every frame.

Fix: Quantize the fontSize in `drawFrameLabelColored` to discrete steps:
```typescript
const rawFontSize = Math.max(4, Math.min(120, 12 / zoom))
const fontSize = Math.round(rawFontSize * 2) / 2  // quantize to 0.5px steps
```

This reduces cache churn from "unique size per zoom level" to "~232 possible sizes" across the full zoom range, dramatically improving cache hit rate during smooth zoom.
